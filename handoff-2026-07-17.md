## 3. 표준 작업 방식 (이 세션 내내 지켜온 규칙)

1. 코드 수정
2. `git commit` (로컬에는 성공)
3. `git push origin claude/current-job-discussion-2d0v4r` 시도 → **항상 403 에러** (권한 없음, 정상적인 현상이니 당황하지 말 것)
4. `/tmp/.../scratchpad/build_preview.py` 실행 → `index.html` + `script.js` + `styles.css`를 하나로 합친 `preview.html` 생성
5. Artifact 도구로 `preview.html`을 아래 **고정 URL**에 재게시 (같은 URL 재사용, 새로 만들지 않음):
   `https://claude.ai/code/artifact/47764b69-c032-4478-8840-9972763f57c2`
6. 바뀐 파일들을 `참택스-최신파일.zip`에도 반영 (스크래치패드에 위치)
7. `SendUserFile`로 바뀐 파일을 사용자에게 직접 전달
8. 사용자가 GitHub 웹 UI에서 수동으로 업로드 (파일명에 하이픈이 있으면 사라지는 업로드 버그가 반복적으로 있었음 — 새 파일은 하이픈 대신 언더스코어 사용 권장, 위 0번 참고)

**주의**: 스크래치패드 디렉토리(`/tmp/claude-0/.../scratchpad`)는 **세션마다 새로 생김**. 이전 세션에서 만든 `build_preview.py`, Playwright 테스트 스크립트들은 새 세션에 없습니다. 필요하면 새로 만들어야 함 (아래 6번에 핵심 스크립트 요약 있음).

## 4. 코드 아키텍처 핵심 패턴

- **다국어 시스템**: `script.js`의 `I18N` 객체에 `{ 'key': { en: '...', zh: '...' } }` 형태로 저장. 한국어는 기본값이라 HTML에 이미 한글로 박혀있고, `data-i18n="key"` 속성이 있으면 `applyTranslations()`가 en/zh일 때만 갈아끼움. HTML 안에 직접 넣기 애매한 동적 텍스트는 `pickLang(ko문자열, en문자열, zh문자열)` 헬퍼로 그때그때 생성.
- **`data-i18n-html`**: 텍스트 안에 `<b>`, `<span>` 같은 태그가 섞여 있을 때 사용 (innerHTML 통째로 교체).
- **`sharedCountry`**: 'kr'/'us'/'cn' — 사용자가 고른 "세금 계산 기준". 홈 화면과 국가비교 화면이 이 값을 공유함.
- **`currentLang`**: 'ko'/'en'/'zh' — 표시 언어. **주의**: localStorage에 저장 안 되고, 새로고침하면 무조건 'ko'로 리셋됨 (단, `?lang=en`/`?lang=zh` URL 파라미터가 있으면 그걸로 시작, 적용 후 URL에서 자동으로 지워짐). 세션 간 언어 기억 기능은 없음(요청받은 적 없어서 안 만듦).
- **FAQ 기준별 필터링** (`data-basis="kr"/"us"/"cn"` 속성): 특정 FAQ 항목이 특정 나라 기준에서만 의미 있을 때(예: 한국 건강보험, 미국 증여세) 이 속성을 붙이면 `filterFaq()`가 `sharedCountry`에 안 맞는 항목을 숨김. 속성 없으면 항상 보임(공통 질문).
- **`FLEX_REF`** (script.js): "이 돈이면 뭘 살 수 있나" 재미 코너에서, 국가 기준(kr/us/cn)별로 다른 참고 가격(강남 아파트/맨해튼 아파트/상하이 아파트, 페라리, 스타벅스 가격)을 쓰도록 만든 테이블. **표시 언어가 아니라 세금 계산 기준을 따라가야 함** — 이 부분이 처음엔 언어 기준으로 잘못 만들어져 있어서 고쳤음.
- **`PAGE_TITLES`** (script.js): 브라우저 탭 제목. `go(view)`로 화면 전환할 때도, `applyTranslations()`로 언어만 바꿀 때도 항상 다시 적용됨. (예전엔 `go()`에서만, 그것도 한국어로 고정되어 있어서 버그였음.)
- **`.flag-badge`** (CSS): 국기 이모지(🇰🇷 등) 대신 쓰는 텍스트 배지(`KR`/`US`/`CN` 등, 초록 배경 흰 글씨). 실제 국기 이모지가 일부 기기에서 "KR" 같은 글자로 깨져 보이는 문제 때문에 사이트 전체에서 국기 이모지를 전부 이걸로 교체함.
- **정적 SEO 페이지들의 언어 라우팅**: `index.html`의 인트로 배너 2번째 카드("해외에 살지만 한국 국적이에요")는 `introAbroadLink`라는 id를 가진 `<a>` 태그이고, `applyTranslations()` 안에서 현재 언어에 맞는 페이지로 href를 바꿔줌 (ko→`korean_abroad_us_lottery_tax_ko.html`, zh→`korean_abroad_us_lottery_tax_zh.html`, en→`korean-abroad-us-lottery-tax.html`).

## 5. 최근에 고친 버그들 (이번 대화 세션 기준, 최신순)

1. **새 파일 2개의 파일명에서 하이픈을 언더스코어로 변경** — GitHub 업로드 시 하이픈이 사라지는 반복적 버그 때문에, 아직 라이브에 없는 파일들만 선제적으로 이름 변경 (0번 참고).
2. **탭 제목이 항상 한국어로 고정되던 버그** + **`?lang=` URL 파라미터가 안 지워져서 새로고침하면 언어가 되돌아가던 버그** — `PAGE_TITLES` 도입, URL에서 `?lang=` 적용 후 `history.replaceState`로 제거하도록 수정.
3. **"해외 거주 한국인" 배너 카드가 언어 상관없이 무조건 영어 페이지로 연결되던 버그** — 한국어판·중국어판 새로 작성, `introAbroadLink` 언어별 분기 추가. 영어판 안의 "한국어로 보기" 링크가 전혀 다른 사람(한국 거주자용)의 페이지로 잘못 연결되어 있던 것도 같이 고침.
4. **FAQ "국민연금·기초연금" 답변에 외국인 관련 설명 누락** — 기초연금은 원칙적으로 대한민국 국적자만 해당된다는 캐비엇 추가 (한국에 사는 외국인 페르소나를 위해).
5. **"이 돈이면 뭘 살 수 있나" 재미 코너가 세금 기준이 아니라 표시 언어를 따라가던 버그** — 중국어로 보면 페라리·스타벅스 가격이 여전히 한국 원화 기준으로 나오던 문제. `FLEX_REF`로 국가 기준별 정확한 참고 가격(미국 페라리 $275,000, 중국 페라리 ¥2.76M, 중국 스타벅스 ¥30 등 — 웹서치로 검증한 값) 반영.
6. FAQ를 세금 계산 기준(kr/us/cn)별로 필터링 + 미국 증여세, 중국 증여세(없음), 각 나라 신고 절차 등 검증된 신규 콘텐츠 추가.
7. 그 외 이번 세션 초반: 캐시 버스팅 버그 수정, 중국 기준 토글 추가, 인트로 배너 카드 전체를 탭 가능하게 변경, 국기 이모지를 `.flag-badge`로 전체 교체, 비교 화면에 머문 채 언어 전환 시 카드가 안 바뀌던 버그 수정 등.

## 6. 검증에 쓴 도구 (스크래치패드, 세션마다 새로 만들어야 함)

- 로컬 정적 서버: `python3 -m http.server 8791` (프로젝트 루트에서)
- Playwright: `chromium.launch({ executablePath: '/opt/pw-browsers/chromium' })` — 이미 설치되어 있어서 `playwright install` 하면 안 됨
- 자주 만들던 테스트 패턴: `setLanguage('ko'/'en'/'zh')`, `setHomeCountry('kr'/'us'/'cn')`, `go('home'/'compare'/'odds'/'faq'/...)` 를 조합해서 순회하며, 화면에 남은 텍스트 중 한글(`/[가-힣]/`) 유출이 있는지, 좁은 화면(240~375px)에서 텍스트가 잘리는지, 언어가 의도치 않게 바뀌는지 체크.
- `node --check script.js` — 문법 오류 없는지 항상 커밋 전에 확인.
