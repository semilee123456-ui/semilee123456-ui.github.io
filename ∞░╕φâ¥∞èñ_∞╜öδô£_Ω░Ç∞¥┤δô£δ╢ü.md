# 참택스(ChamTax) 코드 가이드북

이 문서는 인수인계 문서(`handoff-*.md`, "지금 뭘 했고 다음에 뭘 해야 하는지")와 달리, **이 프로젝트의 디자인/코드 표준이 무엇이고 왜 그렇게 정했는지**를 정리한 참고 문서입니다. 새 기능을 만들거나 기존 걸 고칠 때, "이 사이트에서는 원래 이런 걸 어떻게 처리하나?"를 확인하는 용도로 씁니다.

별도로 디자인팀이 관리하는 `참택스_실행_가이드북_v11.docx`(색채 선호도 연구, 컬러 트렌드, 재물운 상징 등 디자인 리서치 근거)가 있는데, 이 문서는 그것과 별개로 **코드에 이미 반영된 표준을 코드 관점에서** 정리한 것입니다.

---

## 1. 접근성 기준

**대상**: 한국에 거주하는 40~60대. 이 기준 때문에 다른 사이트보다 폰트/명암비 기준이 명시적으로 엄격합니다.

### 최소 폰트 크기
`styles.css`의 `:root`에 정의된 폰트 스케일:
```css
--fs-hero:42px;    /* 결과 숫자 */
--fs-h1:28px;      /* 메인 제목 */
--fs-h2:22px;      /* 큰 섹션 */
--fs-h3:20px;      /* 카드 제목 */
--fs-body:18px;    /* 본문 (시니어 가독성 연구 권장치 반영, 17→18px) */
--fs-note:17px;    /* 설명 */
--fs-small:16px;   /* 최소 (15→16px, 시니어 가독성 연구 최소 권장치) */
--fs-emphasis:25px; /* 카드 강조 숫자 */
```
**`--fs-small`(16px)이 사이트 전체의 절대 최소값입니다.** 이보다 작은 폰트는 배지 안의 국기 코드(`KR`, `US` 등) 정도로만 예외를 허용합니다.

⚠️ **함정**: 42개의 독립된 SEO 랜딩 페이지(`*_in_korea_lottery_tax.html` 등)는 메인 앱과 별개로 자기만의 `:root` 폰트 변수를 갖고 있습니다. 메인 앱의 폰트 기준을 바꿔도 이 페이지들에는 반영되지 않으니, 접근성 관련 변경은 두 곳 다 확인해야 합니다. (2026-07-21에 이 42개 페이지의 `--fs-note`/`--fs-small`이 각각 14px/13px로 기준 미달이었던 걸 발견해서 16px로 통일함.)

### 색상 명암비 (WCAG AA)
본문 텍스트는 배경 대비 최소 4.5:1, 큰 글자(18.66px 이상 굵게)는 3:1을 목표로 합니다. `--status-amber`는 원래 `#B8862E`(명암비 3.00:1)였는데 경고 문구 기준 미달이라 `#946716`으로 더 진하게 조정한 이력이 있습니다 (코드 주석에 남아있음).

**색상 변수를 배경으로 쓸 때 주의**: `--teal`/`--navy`/`--status-red`/`--gold`는 "다크모드에서 어두운 배경 위 글자색으로 쓰일 것"을 기준으로 밝게 튜닝되어 있습니다. 이 색을 반대로 **배경**으로 쓰고 그 위에 흰 글자(`color:#fff`)를 얹으면, 다크모드에서 명암비가 1.2~2.9:1까지 떨어집니다 (밝은 색 배경 + 흰 글자 = 거의 안 보임). 이런 배지/버튼을 만들 때는 반드시 `color:var(--on-accent)`를 쓰세요 — 이 변수는 라이트모드에서 `#fff`, 다크모드에서 잉크색(`#1B1917`)으로 자동 전환됩니다.

## 2. 다국어(i18n) 시스템

### 언어 목록 (22개)
`ko`(기본, HTML에 직접 박혀있음), `en`, `zh`, `vi`, `th`, `ru`(이상 6개는 `pickLang()`의 위치 인자), 그리고 `km`, `ne`, `id`, `my`, `si`, `uz`, `mn`, `kk`, `ky`, `ur`, `bn`, `lo`, `ja`, `ar`, `hi`, `fr`, `tl`(17개, `pickLang()`의 `more` 객체 인자).

`scripts/build-i18n.js`의 `LANGS` 배열과 반드시 일치해야 합니다 — 새 언어를 추가하면 이 배열, `script.js`의 `ADDITIONAL_LANGS`, 그리고 이 문서 둘 다 갱신하세요.

### 두 개의 병렬 시스템
1. **JSON 기반** (`i18n-source/translations.json` → `node scripts/build-i18n.js` → `i18n/*.json`): 정적 HTML 텍스트용. `data-i18n`/`data-i18n-html`/`data-i18n-placeholder`/`data-i18n-aria-label` 속성 + `resolveI18n(key)`.
2. **JS 인라인** (`pickLang(ko, en, zh, vi, th, ru, more)`): 계산 결과나 동적으로 조합되는 문자열용. `more`는 `{ ar:'...', bn:'...', ... }` 형태의 객체.

### 흔한 버그 패턴 (2026-07-21에 대거 발견/수정)
- **`more` 인자를 통째로 빠뜨림** → 17개 언어가 조용히 영어로 폴백. `pickLang()` 호출부에 7번째 인자가 없으면 이 버그입니다.
- **`more` 객체는 있는데 특정 언어(특히 `tl`)만 빠짐** → `tl`이 언어 목록에 나중에 추가돼서, 그 전에 작성된 `more` 객체들에 체계적으로 빠져있었던 이력이 있습니다. 새 `more` 객체를 만들 때는 항상 17개 언어 전부(km, ne, id, my, si, uz, mn, kk, ky, ur, bn, lo, ja, ar, hi, fr, tl) 채우세요.
- **한국어 단위/포맷을 다른 언어에도 그대로 노출** → 예: 확률 분모를 "1 / 1,169만"처럼 한국식 만 단위로 하드코딩해서 영어에도 그대로 보이던 버그. 숫자를 언어별로 다르게 포맷해야 할 땐 `formatWon()`/`formatOddsDenominator()` 같은 언어 분기 헬퍼 패턴을 따르세요.
- **오래된 문구가 안 지워짐** → 기능이 바뀌었는데 번역 문구는 예전 버전 그대로 남는 경우 (예: "한국 vs 미국 비교"였던 게 20개국 비교로 확장됐는데 21개 언어의 제목 문구는 안 바뀌어 있었음, 한국어만 정확했음). **한국어 원문과 다른 언어 번역이 의미상 일치하는지 주기적으로 대조 확인 권장.**

### 검증 스크립트
`pickLang()` 호출 중 `more` 인자가 아예 없는 곳, 있어도 특정 언어 키가 빠진 곳을 찾는 두 개의 감사 스크립트를 만들어 썼습니다 (스크래치패드에 있었음 — 세션마다 새로 만들어야 함, 로직은 `script.js`를 파싱해서 `pickLang(` 호출의 괄호 안 콤마 개수/객체 키를 세는 방식). 새 세션에서 비슷한 감사가 필요하면 이 접근 방식을 재사용하되, **문자열 리터럴 안의 콤마를 따옴표/백틱 경계를 무시하고 세면 오탐이 남으니 주의** (2026-07-21에 이 버그로 51개 vs 73개 카운트가 갈렸던 이력 있음).

## 3. 다크모드

- **opt-in 토글** (자동으로 OS 설정을 따라가지 않음 — 2026-07-21에 이렇게 바꿈). `localStorage.getItem('theme')`, 우상단 토글 버튼 `toggleTheme()`/`syncThemeToggleIcon()`.
- FOUC(깜빡임) 방지: `<head>` 맨 위에 다크모드 여부를 즉시 적용하는 인라인 `<script>` 필요 — 42개 랜딩페이지 전부에 이 스크립트가 있어야 함.
- CSS: `:root[data-theme="dark"]{ ... }`가 라이트모드 `:root{ ... }`의 변수들을 재정의.
- **배경으로 쓰는 색과 글자로 쓰는 색을 구분**할 것 (위 1번 "색상 명암비" 참고, `--on-accent` 패턴).

## 4. 세금 계산 로직 핵심 구조

- `COUNTRY_TAX_PROFILES` (script.js): 20개국의 세금 프로필 배열 (`code`, `flagCode`, `label*`, `implemented`, `needsState`, `more`). 비교 화면 카드와 국가 지도 핀이 이 배열 기준으로 렌더링됩니다.
- `calcTakeHome(eok, countryCode, stateCode)`: 실제 세금 계산. 미국이 먼저 원천징수(30% 또는 24~37%)하고, 거주국이 자국 세율로 추가 계산한 뒤, 외국납부세액공제(FTC)로 `Math.min(미국원천징수액, 자국세액)`을 상계하는 패턴.
- `COUNTRY_TAX_AUTHORITY`/`COUNTRY_TAX_DISCLAIMERS`: 국가별로 "이 계산의 근거가 되는 세무당국" 표기와, 세율이 불확실하거나 특수 사정(조세조약 정지 등)이 있는 나라의 경고 배너 문구.
- **세율 자체나 화면에 노출되는 숫자는 디자인/코드 개선 작업 중에 임의로 바꾸면 안 됩니다** — 이건 프로젝트 초기부터 지켜온 하드 제약입니다. 세율을 바꿔야 하면 반드시 공식 출처를 근거로 남기고 사용자에게 명시적으로 확인받으세요.

## 5. 국가 지도 (2026-07-21 신규)

비교 화면 카드를 탭하면 지도 위 해당 국가 핀이 하이라이트되는 기능. **정확한 국경선이 아니라 장식용으로 단순화한 대륙 모양**입니다 (실제 지리 데이터로 오인되지 않게 하려는 의도).

- `COUNTRY_MAP_COORDS` (script.js): flagCode → `{x, y}` (SVG viewBox `0 0 1000 450` 기준 %). 좌표는 위경도를 등장방형(equirectangular) 도법으로 환산 (`x = (lon+180)/360*100`, `y = (78-lat)/138*100`, 위도 범위는 -60~78로 크롭).
- 새 국가를 추가하면 이 좌표 테이블에도 항목을 추가해야 지도에 핀이 찍힙니다.
- 그룹 카드(실수령액이 같아서 여러 나라가 한 카드로 묶인 경우)는 전체 국기 코드를 나열하지 않고 "N개국 동일" 같은 짧은 문구를 씀 — 국가 수가 많으면 말풍선이 지도 밖으로 넘치는 문제가 있어서 이렇게 바꿨습니다.

## 6. 코드 컨벤션

- **주석은 "왜"를 남긴다**: 이 코드베이스는 "무엇을 하는지"가 아니라 "왜 이렇게 짰는지"(숨겨진 제약, 예전에 있었던 버그, 특정 방식을 선택한 이유)를 한국어 주석으로 남기는 관행이 있습니다. 새 코드를 짤 때도 이 관행을 따르세요.
- **커밋 메시지**: 영어로, "무엇을 왜 고쳤는지" 위주로 씁니다 (한 줄 요약 + 본문에 배경 설명).
- **검증 없이 완료로 표시하지 않는다**: 코드 수정 후 항상 `node --check script.js`(문법), CSS 중괄호 짝 확인, 가능하면 Playwright로 실제 화면을 스크린샷 찍어 확인한 뒤 커밋합니다. 서브에이전트에게 맡긴 작업도 결과를 그대로 믿지 않고 직접 재검증합니다.
