- **이름**: 참택스 (ChamTax)

## 2. 파일/폴더 구조

```
index.html          메인 SPA (홈/비교/확률/FAQ 화면이 전부 이 한 파일 안에 .view로 존재)
styles.css           전체 스타일 (디자인 토큰은 :root, 다크모드는 :root[data-theme="dark"])
script.js            전체 로직 (세금 계산 + 렌더링 + i18n — 800KB+, 매우 큰 파일)
manifest.json        PWA 매니페스트
sitemap.xml          SEO 사이트맵 (42개 URL, lastmod는 실제 변경일 기준으로 유지)

i18n/                언어별 번역 JSON (22개 파일) — 여기 직접 고치면 안 됨, 빌드 산출물임
i18n-source/
  translations.json  번역 원본(source of truth) — 여기를 수정
scripts/
  build-i18n.js       translations.json → i18n/*.json 생성 스크립트

tests/               Playwright 회귀 테스트 3개 (아래 7번 참고)

*.html (그 외 42개)   언어별/국가별 SEO 랜딩 페이지 — 메인 계산기와 완전히 별개, 각자
                      <style> 태그 안에 자체 :root 색상/폰트 변수를 중복 정의하고 있음
                      (이게 반복적으로 버그 원인이 됨 — 아래 6번 참고)
```

### 다국어(i18n) 작업 방법
1. `i18n-source/translations.json`에서 해당 키의 번역을 수정 (**절대 `i18n/*.json`을 직접 고치지 말 것** — 다음 빌드 때 덮어써짐)
2. `node scripts/build-i18n.js` 실행 → `i18n/*.json` 22개 재생성
3. HTML에 박힌 정적 텍스트: `data-i18n`/`data-i18n-html`/`data-i18n-placeholder`/`data-i18n-aria-label` 속성 + `resolveI18n(key)`가 `i18n/{lang}.json`을 fetch해서 치환
4. JS에서 만드는 동적 문자열: `pickLang(ko, en, zh, vi, th, ru, more)` — 6개는 위치 인자, 나머지 17개 언어는 `more` 객체. **`more`를 빠뜨리면 그 17개 언어에서 영어로 조용히 폴백됨** — 이게 오늘 세션에서 발견한 버그의 대다수 원인이었음 (아래 8번 참고)

## 4. 코드 아키텍처 핵심 패턴

- **다국어**: 위 2번 참고. JSON 기반(`data-i18n*`)과 JS 인라인(`pickLang`) 두 시스템이 공존함 — 새 텍스트를 추가할 때 어느 쪽에 넣을지 헷갈리기 쉬운데, **정적 HTML 텍스트는 JSON, 계산 결과/동적 조합 텍스트는 pickLang**이 일반적인 구분 기준
- **다크모드**: `:root[data-theme="dark"]`, `localStorage.getItem('theme')`, `toggleTheme()`/`syncThemeToggleIcon()` (script.js). **색상 변수를 배경으로 쓸지 글자색으로 쓸지에 따라 명암비가 달라짐** — 새 UI를 만들 때 `--teal`/`--status-red`/`--gold`/`--navy`를 배경으로 쓰고 흰 글자를 얹는다면 반드시 `color:var(--on-accent)`를 쓸 것 (하드코딩된 `color:#fff` 금지)
- **접근성 폰트 floor**: `styles.css`의 `--fs-small:16px`가 명시적 최소값(주석에 "시니어 가독성 연구 최소 권장치"라고 근거 남겨둠). 랜딩페이지들은 **자체 CSS 변수를 따로 갖고 있어서** 메인 앱과 별도로 체크해야 함
- **`COUNTRY_TAX_PROFILES`** (script.js): 20개국의 세금 프로필 배열. 비교 화면 카드, 국가 지도 핀(`COUNTRY_MAP_COORDS`, flagCode로 매칭)이 이 배열을 기준으로 렌더링됨. 새 국가를 추가하려면 여기에 항목 추가 + `calcTakeHome()`에 해당 국가 세율 로직 + 지도 좌표까지 3곳을 같이 갱신해야 함
- **`formatWon(n)`**: 원화 금액 표시를 언어별로 분기하는 핵심 함수 (억/조 vs million/billion). 새로운 "숫자를 언어별로 다르게 표시해야 하는" 요구가 생기면 이 함수의 분기 패턴을 참고 (`formatOddsDenominator()`도 같은 패턴으로 오늘 새로 만듦)
- **자체완결 프리뷰 빌드**: `<link rel="stylesheet" href="styles.css...">`와 `<script src="script.js...">`를 각각 `<style>`/`<script>` 인라인으로 치환하는 간단한 Python 스크립트로 만듦 (아래 6번 참고)

## 6. 자체완결 프리뷰 만드는 법 (스크래치패드용, 매 세션 새로 필요)

```python
import re
html = open('index.html', encoding='utf-8').read()
css = open('styles.css', encoding='utf-8').read()
js = open('script.js', encoding='utf-8').read()
html2 = re.sub(r'<link rel="stylesheet" href="styles\.css[^"]*">', lambda m: '<style>\n' + css + '\n</style>', html)
html2 = re.sub(r'<script src="script\.js[^"]*"[^>]*></script>', lambda m: '<script>\n' + js + '\n</script>', html2)
open('preview.html', 'w', encoding='utf-8').write(html2)
```

## 7. 알려진 남은 이슈 / 손 안 댄 부분 (의도적으로 남겨둠)

- `LANGUAGE_CONTENT_PAGES`의 aria-label(23개 항목)이 영어/한국어 폴백 상태 — 화면에 보이는 링크 텍스트는 각 언어 원문인데, 스크린리더용 aria-label만 완전 현지화하려면 약 400개 문자열이 더 필요해서 비용 대비 효과 판단상 보류함
- 미국 주(State) 이름은 모든 언어에서 영어 그대로 노출됨 (의도적 — 고유명사라 번역이 오히려 혼란을 줄 수 있다고 판단)
- `odds.howtoText`(파워볼 설명 텍스트)는 아직 6개 핵심 언어만 지원 — 기존 코드 주석에 "이건 새 콘텐츠라 우선 6개 언어만" 이라고 명시되어 있음, 확장하려면 나머지 17개 언어 번역 추가 필요
- 국가 지도는 정확한 국경선이 아닌 장식용 단순화 도형임 — 실제 지리 정보로 오인되지 않게 하는 게 의도이니, 지도 정확도를 높여달라는 요청이 오면 이 설계 의도부터 확인할 것
